The 7 layers of the OSI (Open Systems Interconnection) model standardize network communication from user application to physical cable. They are: ==(7) Application, (6) Presentation, (5) Session, (4) Transport, (3) Network, (2) Data Link, and (1) Physical==. Data travels from Layer 7 down to Layer 1, then up to Layer 7 on the receiving end.

## **The 7 Layers of the OSI Model**

- **7. Application Layer:** The layer closest to the end-user; provides network services to applications (e.g., HTTP, FTP, SMTP, DNS).
- **6. Presentation Layer:** Formats, translates, encrypts, and compresses data to ensure it is readable by the application layer.
- **5. Session Layer:** Establishes, manages, maintains, and terminates connections (sessions) between local and remote applications.
- **4. Transport Layer:** Handles end-to-end communication, flow control, segmentation, and error checking (e.g., TCP, UDP).
- **3. Network Layer:** Determines the physical path for data (routing) and handles logical addressing (IP addresses).
- **2. Data Link Layer:** Responsible for node-to-node data transfer, error detection/correction, and MAC addressing. It formats data into frames.
- **1. Physical Layer:** Transmits raw data as physical signals (electrical signals, light pulses, or radio waves) over physical mediums.

**Why the OSI Model Matters**  
This conceptual framework simplifies troubleshooting, promotes vendor interoperability, and helps define network protocols.
